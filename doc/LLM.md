## Summary

ChatGPT was used as a development assistant during the implementation of the TopWords project.

## How It Was Used

The LLM provided help with:

- Clarifying project architecture (separating `core` and `app`)
- Designing the sliding window logic and observer pattern
- Setting up command-line parsing using `mainargs`
- Debugging SBT configuration and runtime issues
- Suggesting unit test structure and identifying edge cases

## Verification Process

All suggestions generated by the LLM were manually reviewed, adapted where necessary, compiled, and tested locally.  
Final implementation decisions and validation were performed independently.

## Extra Credit Implemented

The following features were added to go beyond the base assignment requirements:

1. **Ignore List for Words**   
   - Implemented functionality to read a list of words from a file that are always ignored in the word cloud, regardless of their length.

2. **Case-Insensitive Word Counting**   
   - Words are treated case-insensitively. For example, `Apple` and `apple` are counted together.

3. **Update Word Cloud Every k Steps** 
   - Added a command-line argument `--every-k-steps` to only update the word cloud after processing every `k` words, improving performance for large streams.

4. **Minimum Frequency Threshold** 
   - Added a command-line argument `--min-frequency` to include only words that occur at least `n` times in the word cloud.

---

## Notes
- Each of these features was integrated into `WordStreamProcessor.scala` and tested in `WordStreamProcessorTest.scala`.
- The LLM assisted with Scala 3 syntax, proper mutable collections usage, and MUnit test creation.


## Conclusion

The LLM was used strictly as a support tool for guidance and debugging.  
All final code, integration, and testing were completed manually.
